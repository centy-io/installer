# Installer builds incorrect asset download URLs (missing v prefix)

## Bug

In `lib/src/github.rs:57`, `release_info()` strips the `v` prefix from the version tag when constructing asset download URLs:

````rust
let version_num = tag.strip_prefix('v').unwrap_or(tag);
let asset_name = format!("centy-daemon-{version_num}-{}{}", platform.target, platform.archive_ext);
````

This produces asset names like `centy-daemon-0.1.6-x86_64-unknown-linux-gnu.tar.gz`.

However, actual GitHub release assets include the `v` prefix:

* `centy-daemon-v0.1.6-x86_64-unknown-linux-gnu.tar.gz`
* `centy-daemon-v0.1.6-aarch64-apple-darwin.tar.gz`
* `centy-daemon-v0.1.6-x86_64-pc-windows-msvc.zip`

## Impact

All downloads fail with 404 because the constructed URLs donâ€™t match actual asset names.

## Fix

Remove the `strip_prefix('v')` call and use the tag as-is in the asset name format string. Update the unit test `release_info_builds_urls` accordingly.
